HTML

  • Hypertext Markup Language


    HTML provides the structure of a webpage. It doesn't provide any info about how the components look like (color or size). HTML tells you only what are the components.

    In summary HTML anotates content and defines document structure.

    <p> - paragraph tag

  • History of HTML


    W3C - the first organisation that was developing HTML.

    1997 - it was wild west of the web.

    2000 - W3C wanted XHTML 1.0 (wrong direction).

    2004-WHATWG (browsers bending together) started driving HTML.

    2007-WHATWG and W3C started working together (and produced HTML5).


    Modern Browsers silently update themself without asking the user for it. (evergreen browsers- all modern browsers - Chrome, Firefox, Edge).

    IMPORTANT WEB LINKS:

    www.caniuse.com - information about which browser supports what feature.

    validator.w3.org - validates html code or webpage.

    w3schools.com Statistics about browser usage.

  • Anatomy of HTML tag


    ELEMENT NAME---><opening tag>CONTENT</closing tag>


    No space is allowed to exist between opening bracket and element name also between closing bracket and the slash.

    Space is allowed everywhere else and extra space gets ignored.


    <p id="myId"></p> id-attribute name, "myid" -attribute value.


    Attributes can only be specified on the opening tag.

    Double quotes and single quotes needs to be closed in proper order (double single single double).

    In HTML 5 selfclosing tags are removed even empty tag should have opening and closing tag.

  • Basic HTML document structure


    <!doctype html> - HTML5 declaration THIS TAG ALWAYS HAS TO BE USED (starting the html document).

    <!doctype html><html><head><meta><title></title></head><body></body></html> - usuall basic structure of HTML file.

    <!--Comment text is here--> - comment in html.


    Important rule is that if 2 tags are nested you cannot close the parent before closing the child.

    When browser opens the page it interpret the html code sequentially from top to bottom. 1. doctype 2. html 3. head etc.

  • HTML Content Models

    <div> - generic BLOCK element

    <span> - generic INLINE element


    Block elements are not allowed to be child of inline elements.

    Block elements start on new row while nested inline elements on same row.

    There is scheme and list of the types of content, tags (flow, phrasing): www.w3.org/TR/html/dom.html#the-nothing-content-model

    Officially not part of HTML5, but still used.

    Roughly equivalent to flow content & phrasing content.

  • Heading elements and semantic tags


    <h1>SEO specialists agree that this heading is the most important heading and helps SEO ranking</h1>

    Heading tags in html should not be used for text size. Everything can be applied using CSS.


    Semantic tags:

    <header><nav></nav></header> - HTML5 tag that can consist a company logo, tag line or navigation. Navigation usually contains links to different parts of the web site.

    <section><article></article></section> - article usually should go inside a section this way the html code is easier to read.

    <aside> - is basically content that is related to the main topic of the page but not in direct relationship as the main content.

    <footer> - footer information is within this tag


    Semantic HTML elements make the code easier to read.

    Semantic HTML elements are block level elements(new line).

    Well chosen content of H1 element is crucial to SEO.

    Semantic elements allow for a more meaningful expression of the structure of our HTML page.

  • Lists


    <ul> - unordered list with another nested <ul> child you can create a sublist.

    <li> - every element of a list.

    The only thing that is allowed in UL element is LI element everything else is not allowed.


    <ol> - ordered list you can create sublist same way as with unordered list


    Lists provide a natural and commonly used grouping of content.

    Very often, lists are used for structuring navigation portions of the web page.

  • HTML Character Entity References


    &amp; - & and symbol.

    &lt; - < less than symbol.

    &gt; - > greater then symbol.

    &copy; - © copyright symbol.

    &nbsp; - placing it between words make the text staying on the same row no matter what.

    &quot; - quote unrestricted by character set (using normal quotes can display ugly symbols with certain charsets).


    Html Entities help avoid rendering issues.

    Safeguard against more limited character encoding.

    Provide characters not available on a keyboard.

  • Creating Links


    External Links

    <a href="http://www.google.com" target="_blank" title="Like Our Page!"> - external link tag

    target="_blank" - opens link in new window.

    title="" - helps accessibility people (text to speech).


    Internal Links

    <a href="#section1">#section1</a> - internal link used for navigation to section1 tag

    <section id="section1"> - notice that section id does not have # before its name

    This type of links links to element by its id.

    Navigation links are very important for SPA (single page applications).


    A tag is not only inline tag (as of HTML5) this is a way to make company logo clickable as homepage. This is very easy to do since HTML5.

    <a href="same-directory.html" title="same dir link">

    <div> DIV linking to a file in the same directory</div>

    </a>

  • Displaying Images


    <img src="picture.jpg" width="400" height="235" alt ="Picture with a quote">

    Its good practice to allways set height and width of the image because this way the browser

    is reserving the space for an image and website layout stays same even if the image doesnt load.

    Alt tag is helping accessibility to people that use text to speech.
