  • Anatomy of HTML tag


    ELEMENT NAME---><opening tag>CONTENT</closing tag>


    No space is allowed to exist between opening bracket and element name also between closing bracket and the slash.

    Space is allowed everywhere else and extra space gets ignored.


    <p id="myId"></p> id-attribute name, "myid" -attribute value.


    Attributes can only be specified on the opening tag.

    Double quotes and single quotes needs to be closed in proper order (double single single double).

    In HTML 5 selfclosing tags are removed even empty tag should have opening and closing tag.

  • Basic HTML document structure


    <!doctype html> - HTML5 declaration THIS TAG ALWAYS HAS TO BE USED (starting the html document).

    <!doctype html><html><head><meta><title></title></head><body></body></html> - usuall basic structure of HTML file.

    <!--Comment text is here--> - comment in html.


    Important rule is that if 2 tags are nested you cannot close the parent before closing the child.

    When browser opens the page it interpret the html code sequentially from top to bottom. 1. doctype 2. html 3. head etc.

CSS

  • Cascading Style Sheets Basics


    Power of CSS

    On the web content is the king. How users engage with the content and how the content is presented is actually very important.

    The use of color, positioning, size, among other things, is all part of that. Cascading style sheets, or CSS, is the technology that provides the styling capability.


    www.csszengarden.com - same content presented in hundreds of web designs


    CSS is an incredibly powerful technology that can not only style your content, but also also create a particular user experience that you would want your users to have.

    Anatomy of CSS rule


    Selector --->p {

    color: blue;<--declaration, color is the property, blue is the value

    }


    A CSS rule can consist several declarations.

    A collection of CSS rules is called a stylesheet.

    Most of the browsers come with predefined default styles. (its our job to redefine or modify theese).

    To make a comment in CSS file we can use: /*comment*/

    Element, Class and ID Selectors


    Element selector

    p {

    color: blue;

    }


    Every p element in our html page is going to be affected by this CSS rule.


    Class selector

    definition of a class--->.blue {

    color: blue;

    }

    <p class="blue">...</p> - usage of a class in html tag.


    Class selector is specified by dot followed by the name of the class.

    To be used the html tags should have the property class="blue"(example).

    For example the default paragraphs will be completely unaffected by this CSS rule.

    There is difference in syntax between declaration and usage of class selectors.


    ID selector

    definition of a id--->#name {

    color: blue;

    }

    <div id="name">...</div> - usage of id in html tag.

    In this example every html tag that is not with the id name will be completely unaffected by this CSS rule.

    Id selectors can't and should not be reused (their use is to style a specific element).


    CSS declarations can be grouped(CSS rules will apply to multiple selectors).

    Example of grouped CSS rule for div selectors and blue class. Notice that selectors are comma separated:

    div, .blue {

    color: blue;

    }

    Combining Selectors


    Element with Class Selector

    The following example targets every p that has class="big":

    p.big {

    font-size: 20px;

    }


    There should not be any spaces between the first selector and the second selector.

    Only "p" tags with class="big" will follow this CSS rule. Any other tags with the same class will just ignore it.


    Child Selector

    article > p {

    color: blue;

    }


    Only "p" elements that will be affected are going to be the one that are child of article elements.

    "p" elements that are child of other elements (div,section) wont be affected. This selector is not limited to Element Selectors (can be used in combination with class or id selector).


    Descendant Selector

    article p {

    color: blue;

    }


    This selector is targeting every p element that is inside (at any level) of article element. It will affect elements even if they are not direct child of article element. Example:

    <article><div><p></p></div></article> - this p will be affected by this selector

    This selector is not limited to Element Selectors (can be used in combination with class or id selector).


    Adjacent sibling selector (selector + selector) less used generally so not covered.

    General sibling selector (selector ~ selector) less used generally so not covered.

    Pseudo-Class Selectors


    selector:pesudo-class {

    code

    }


    a:link, a:visited {

    text-decoration: none; - removing underline of hyperlinks in this case

    background-color: green;

    border: 1px solid blue;

    color: black; - changing text color

    display: block;

    width: 200px;

    text-align: center;

    margin-bottom: 1px;

    }


    :link and :visited Pseudo-Class Selectors are controlling the style of hyperlinks before and after they are visited.


    a:hover, a:active {

    background-color: red;

    }


    :hover and :active Pseudo-Class Selectors are controlling the style when hovering over the link and in the moment of clicking it.


    The code in the following example will make bigger the 3rd link of the list:

    header li:nth-child(3) {

    font-size: 24px;

    }


    The code in the following example will make every second element with gray background (good for tables):

    section div:nth-child(odd) {

    background-color: gray;

    }


    As selectors can be combined pseudo selectors can also be combined(will change the cursor ever the 4th element):

    section div:nth-child(4):hover {

    cursor: pointer;

    }


    Simple/readable is preferred instead of Complicated/Tricky.

JavaScript

  • Basics of Javascript


    Where to place Javascript Code

    <script<>/script> - inline script

    <script src="js/script.js"></script> - external file script

    console.log(); - function thats available on all browsers (except internet explorer)


    Javascript is a single threaded enviroment.

    Javascript gets executed sequentially (as of html).


    Defining Variables, Function and Scope

    Variables

    Variable definition should always start with 'var'.

    In JS no types are daclared. Javascript is dynamically typed language. Same variable can hold different types during the life of the execution.

    var x = 4, y = 4; - this is shortcut in JS for multiple declarations

    Functions

    Function a () {...} - usuall way to define function

    var a = function () {...} - no name is defined after the function keyword

    In both ways we execute the function with:

    a(); - function execution/invocation (synonyms)


    We can also define arguments to be passed to the function (arguments are defined without 'var'):

    function compare (x, y) {

    return x > y;

    }


    Everything below is perfectly legal synthax in JS:

    function compare (x, y) {...}

    var a = compare(4,5); - after execution the result will be stored in a

    compare(4, "a"); - we are not storing the result anywhere

    compare(); - we are executing the function without passing any value

    Scope

    Global scope - variables and functions defined here are available everywhere. Global scope does not have an Outer Environment as it's the most outer there is.

    Function aka lexical - variables and functions defined here are available only within this function

    Scope chain - everything is executed in an Execution Context. Function invocation creates a new Execution Context.


    Each Execution Context has:

    -its own Variable Environment

    -Special 'this' object

    -Reference to its Outer Environment


    Example of how scope chain work:

    Referenced (not defined) variable will be searched for in its current scope first. If not found, the Outer Reference will be searched. If not found, the Outer Reference's Outer Reference will be searched, etc. This will keep going until the Global Scope. If not found in Global scope, the variable is undefined(not found).


    Another code example of how scope chain work:

    var message = "in global";

    console.log("global: message = " + message); - The message is stored in this variable which is also stored in the Window object(global execution context).


    var a = function () {

    var message = "inside a";

    console.log("a: message = " + message);

    b(); - output will be b: message inside global scope

    }


    Function b () {

    console.log("b: message = " + message);

    }


    a(); - Output will be a: message = inside a

    If we define the function b inside function a - output of function b will be inside a.


    In summary:

    All variables are dynamically typed.

    Defining functions - creates its own scope (lexical).

    JS core runs within an Execution Context.

    Scope chain is used to retrieve variables from Outer Variable Environments.

  • Javascript Types and Common Language Constructs


    Each language defines some build-in types. Build-in types can be used to build other data structures.


    Object type

    Object type is a collection of name/value pairs. In below example favorites is name. Everything inside the curly braces is the value of this pair:

    Person Object:

    firstName: "Velizar",

    lastname: "Dobrev",

    favorites: {

    Burger: "MC Cheeseburger"

    Candy: "Green Snickers"

    IceCream: "White Boss"

    }


    Primitives

    Primitive type represents a single, immutable value (not an object). Immutable means once it's set, it can't be changed. Value becomes read-only.

    1) Boolean can only have 2 values: true or false.

    2) Undefined signifies that novalue has ever been set. Can only have one value: undefined. A variable should never be set to undefined because this is counter to the core meaning of undefined.

    if (x == undefined) {

    console.log("x is undefined");

    }

    Undefined means it has been declared. But not given any value.

    var x;

    The memory has been allocated but not value has been stored in this memory space (no value has ever been explicitly set yet).

    3) Another value that JS language defines is NULL. Null signifies the lack of value (opposed to undefined- lack of definition). It's ok to explicitly set a variable to null.

    4) Number is the only numeric type in JavaScript. It's always represented under the hood as double-precision 65-bit floating point. JS does not have an integer type. Integers are a subset of doubles instead of a separate data type.

    5) String is sequence of characters used to represent text.

    6) Symbol is new to ES6 and is not covered.


    In summary:

    Javascript defines 7 build-in types. One object and 6 Primitives.

  • Programming Theory


    OOP Polymorphism:


    Wikipedia: Polymorphism is a programming language feature that allows values of different data types to be handled using a uniform interface.
    Another short definition: The ability of objects of one type to have one and the same interface, but different implementation of this interface.


    Example: Fruit can be eaten, as a general rule, but different types of fruit is eaten in different ways. An apple, which is a fruit, can be eaten (because it is a fruit). A banana can also be eaten (because it is also a fruit), but in a different manner from an apple. You peal it first. This illustrates inheritance (fruit can be eaten), polymorphism (something that eats fruit can eat all types of fruit), and encapsulation (a banana has a skin).
